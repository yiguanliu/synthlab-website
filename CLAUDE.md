## What this is

Marketing website for Synthlab (synthlab.io) — a company producing bespoke synthetic datasets for AI/ML training, currently focused on manufacturing fault and edge-case detection. Static site: vanilla HTML/CSS/JS, no framework, no build step, no dependencies.

The site content is derived from the pitch deck at `../Synthlab_Pitch_Deck.pptx` (generated by `../build_deck.js`); the company brief lives at `../BRIEF.md`. Keep website copy consistent with those sources when editing.

## Architecture

Single page (`index.html`) styled by `styles.css`, with three small script files loaded at the end of `<body>` in this order:

- **`icons.js`** — inline lucide-style SVG icon set as template strings. Icons are injected into any element with a `data-icon="name"` attribute by `injectIcons()` on DOMContentLoaded. To add an icon, add an entry to the `ICONS` map; icons use `stroke="currentColor"` so they inherit text color.
- **`animations.js`** — IntersectionObserver scroll-reveal for `[data-reveal]` elements (staggered via `data-reveal-delay="1|2|3"`), plus an auto-cycling step highlighter for `[data-diagram]` / `[data-diagram-step]` markup.
- **`chart.js`** — two canvas animations: the hero oscilloscope chart (`#hero-chart`, three damped-oscillation waves with legend, axes, and an auto-sweeping crosshair that follows the mouse on hover) and a simpler looping sine wave (`#approach-chart`, only rendered if the element exists). Colors are hardcoded constants at the top of each IIFE — when changing the site theme, update them there too; they don't read CSS variables.
- **`script.js`** — footer year and smooth anchor scrolling.

## Design system

Black-and-white editorial style. All theming flows from CSS variables in `:root` in `styles.css`: Georgia serif for headings (`--serif`), system sans for body (`--sans`), monospace for eyebrows/labels/nav/buttons (`--mono`). Layout grids are drawn with 1px hairline borders (no shadows, no rounded corners). Dark sections use the `.section-dark` class with `--ink-dark`/`--muted-dark`/`--line-dark` counterparts.

Conventions to preserve:
- No em-dashes in site copy (user requirement; use periods, commas, or colons).
- Showcase images keep their native aspect ratio (`width: 100%; height: auto;`) — never crop with `object-fit`.
- All animations must respect `prefers-reduced-motion` (canvas charts bail out entirely; CSS reveals are disabled via media query).
- Mobile breakpoints: 1024px (grids to 2-col), 860px (single column, hero chart hidden), 640px (full mobile with border-artefact fixes on collapsed grids).
